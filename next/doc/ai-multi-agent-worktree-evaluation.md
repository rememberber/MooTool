# Multi-Agent / Git Worktree 稳定版评估

评估日期：2026-07-18  
目标平台：macOS x64（Intel MacBook）  
结论：**暂不进入稳定版；保留为后续实验能力。**

## 已具备的基础

- Agent Profile 已绑定客户端、工作目录、权限模式、模型运行时和配置指纹。
- Codex/Claude 受控 CLI 任务使用无 Shell 子进程、Prompt stdin、超时、取消、输出上限和进程树清理。
- 所有会修改 AI 配置的功能已有计划、显式批准、快照、验证和回滚。
- Prompt Lab 可在进入自动编排前独立评估模型输出质量与 Token 成本。

## 稳定版门槛

| 门槛 | 当前状态 | 进入稳定版前需要的证据 |
| --- | --- | --- |
| 每个 Agent 独占 Worktree 与分支 | 未实现 | 路径防逃逸、分支保留、并发创建和重复恢复测试 |
| 权限按 Agent 隔离 | 部分具备 | MCP/Skill/环境变量按任务生成最小权限快照，任务中不可扩权 |
| Git 操作可恢复 | 未实现 | 创建失败、CLI 中断、App 崩溃、未提交修改和 Worktree 锁的恢复测试 |
| 冲突不自动覆盖 | 未实现 | 三方差异、二进制文件、重命名、删除/修改冲突的只读汇总与人工决策 UI |
| 依赖与并发有界 | 未实现 | DAG 校验、循环依赖拒绝、全局/项目并发上限、取消传播 |
| 结果可归因 | 部分具备 | 每个 Agent 的提交、文件差异、测试结果、Token/耗时与 Profile 指纹绑定 |
| 外部副作用受控 | 已有原则 | 禁止默认 push、发布、发消息和跨项目 Shell；新增动作必须单独授权 |
| 清理安全 | 未实现 | 仅清理 MooTool 创建且已验证无未提交内容的 Worktree；不使用宽泛递归删除 |

## 主要风险

1. 原生客户端对 Subagent、权限提示和会话恢复的语义不同，不能伪装为统一且可预测的 Runtime。
2. `git worktree remove --force` 会让未归档修改难以恢复，因此不能作为失败后的默认清理策略。
3. 多 Agent 同时修改共享配置、锁文件或生成物时，即使文件级无冲突，也可能产生语义冲突。
4. CLI 成功退出不等于任务质量达标；稳定版必须把测试、质量阈值与成本门槛纳入完成条件。
5. Intel MacBook 可放宽吞吐时间，但不能放宽取消响应、内存上限、权限确认和恢复正确性。

## 推荐实验切片

首个实验版本只允许用户选择一个干净的 Git 项目，并创建两个**只读或 workspace-write、禁止网络副作用**的任务。MooTool 只负责：

1. 预览将创建的分支和 Worktree 路径；
2. 用户逐项批准后创建；
3. 在各自 Worktree 中运行现有受控 CLI 任务；
4. 展示提交、未提交 Diff、测试结果和冲突，不自动合并；
5. 仅在 Worktree 干净且引用可验证时允许清理。

在上述门槛全部有自动化故障注入测试之前，多 Agent/Worktree 不注册为稳定工具，也不在后台自动创建分支或目录。
